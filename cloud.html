Panduan Praktis Google Cloud Platform

Bagian 1: Pengenalan Cloud Computing

Apa itu Cloud Computing?

Cloud computing adalah layanan komputasi (server, storage, database, networking, software) yang diakses via internet dengan model pay-as-you-go.

Keuntungan Cloud Computing

  • Tidak perlu beli & maintain server fisik
  • Skalabilitas mudah (naik-turunkan resource sesuai kebutuhan)
  • Bayar hanya yang dipakai
  • Akses dari mana saja
  • High availability & disaster recovery

Model Layanan Cloud

  • IaaS (Infrastructure as a Service) - Sewa VM, storage, network
  • PaaS (Platform as a Service) - Platform untuk deploy app tanpa kelola infrastruktur
  • SaaS (Software as a Service) - Aplikasi siap pakai lewat browser

Google Cloud Platform (GCP)

GCP adalah platform cloud dari Google yang menyediakan 100+ services untuk compute, storage, database, AI/ML, dan lainnya.

Layanan GCP yang Populer:

  • Compute Engine - Virtual machines
  • Cloud Run - Container serverless (kita pakai ini)
  • App Engine - PaaS untuk web apps
  • Cloud Storage - Object storage
  • Cloud Functions - Serverless functions
  • Cloud SQL - Managed database

Bagian 2: Setup Akun Google Cloud

Langkah 1: Buat Akun Google Cloud

  1. Buka browser dan kunjungi: 
    https://cloud.google.com/
  2. Klik Get started for free atau Try it free
  3. Login dengan akun Google Anda (atau buat baru)
  4. Pilih negara Indonesia
  5. Setujui Terms of Service

Langkah 2: Verifikasi Akun & Free Trial

  1. Isi informasi kartu kredit/debit untuk verifikasi
    Catatan: Google memberikan $300 free credit untuk 90 hari. Anda tidak akan dicharge selama masih free trial, dan harus upgrade manual ke paid account.
  2. Lengkapi survey singkat tentang penggunaan GCP
  3. Klik Start my free trial
  4. Tunggu proses aktivasi (1-2 menit)

Langkah 3: Buka Google Cloud Console

  1. Setelah aktivasi selesai, Anda akan diarahkan ke Console
  2. URL Console: 
    https://console.cloud.google.com/
  3. Bookmark URL ini untuk akses cepat

Bagian 3: Mengenal Google Cloud Console

Dashboard Console

Ketika pertama kali login, Anda akan melihat:

  • Navigation Menu (‚ò∞) - Menu utama di kiri atas
  • Project Selector - Dropdown untuk pilih/ganti project
  • Search Bar - Cari services atau resources
  • Cloud Shell - Terminal di browser (icon >_)
  • Notifications - Bell icon untuk notifikasi
  • Account Menu - Info akun dan billing

Langkah 1: Buat Project Baru

  1. Klik Project Selector di top bar
  2. Klik NEW PROJECT
  3. Isi nama project: 
    my-first-cloud-project
  4. Biarkan Location default (No organization)
  5. Klik CREATE
  6. Tunggu project dibuat (~30 detik)
  7. Pilih project yang baru dibuat dari dropdown
Penting: Setiap project di GCP punya Project ID unik. Project ID ini akan digunakan di command line dan API calls.

Langkah 2: Enable Billing untuk Project

  1. Buka menu ‚ò∞ ‚Üí Billing
  2. Pilih project Anda
  3. Link billing account (free trial) ke project
  4. Verifikasi bahwa billing sudah active

Langkah 3: Eksplorasi Navigation Menu

Buka menu ‚ò∞ dan lihat kategori services:

  • Compute - VM, containers, serverless
  • Storage - Cloud Storage, Filestore
  • Databases - SQL, NoSQL, caching
  • Networking - VPC, Load Balancing, CDN
  • Operations - Logging, Monitoring, Debugging

Bagian 4: Install Google Cloud CLI (gcloud)

Langkah 1: Download & Install gcloud CLI

  1. Kunjungi: 
    https://cloud.google.com/sdk/docs/install
  2. Download installer untuk Windows
  3. Jalankan installer (GoogleCloudSDKInstaller.exe)
  4. Ikuti wizard instalasi dengan setting default
  5. Centang opsi "Run gcloud init" di akhir instalasi

Langkah 2: Initialize gcloud

  1. Setelah instalasi, akan terbuka Command Prompt dengan gcloud init
  2. Atau buka Command Prompt baru dan jalankan: 
    gcloud init
  3. Pilih Log in with a new account
  4. Browser akan terbuka untuk login
  5. Login dengan akun Google yang sama dengan GCP
  6. Authorize gcloud untuk akses akun Anda
  7. Kembali ke terminal, pilih project yang sudah dibuat
  8. Pilih default compute region (pilih yang terdekat, misalnya asia-southeast1 untuk Singapore)

Langkah 3: Verifikasi Instalasi gcloud

  1. Cek versi gcloud: 
    gcloud --version
  2. Cek konfigurasi: 
    gcloud config list
  3. Cek project aktif: 
    gcloud config get-value project

Bagian 5: Deploy Aplikasi Pertama ke Cloud Run

Tentang Cloud Run

Cloud Run adalah layanan serverless untuk deploy container. Keuntungannya:

  • Tidak perlu kelola server
  • Auto-scaling (0 sampai banyak instance)
  • Bayar per request & compute time
  • Support semua bahasa pemrograman (via Docker)

Langkah 1: Buat Aplikasi Web Sederhana

  1. Buat folder project: 
    mkdir hello-cloud
cd hello-cloud
  2. Buat file app.py dengan isi: 
    from flask import Flask
import os

app = Flask(__name__)

@app.route('/')
def hello():
    return '''
    <html>
        <body>
            <h1>Hello from Cloud Run!</h1>
            <p>Aplikasi ini berjalan di Google Cloud.</p>
        </body>
    </html>
    '''

if __name__ == '__main__':
    port = int(os.environ.get('PORT', 8080))
    app.run(host='0.0.0.0', port=port)
  3. Buat file requirements.txt: 
    Flask==3.0.0

Langkah 2: Buat Dockerfile

  1. Buat file bernama Dockerfile (tanpa extension) di folder yang sama: 
    # Use official Python runtime
FROM python:3.11-slim

# Set working directory
WORKDIR /app

# Copy files
COPY requirements.txt .
COPY app.py .

# Install dependencies
RUN pip install --no-cache-dir -r requirements.txt

# Run the application
CMD ["python", "app.py"]

Langkah 3: Enable Cloud Run API

  1. Jalankan command: 
    gcloud services enable run.googleapis.com
  2. Tunggu proses enable selesai (~1 menit)

Langkah 4: Deploy ke Cloud Run

  1. Jalankan command deploy: 
    gcloud run deploy hello-cloud --source . --region=asia-southeast1 --allow-unauthenticated
  2. Penjelasan command:
    • hello-cloud - nama service
    • --source . - deploy dari folder saat ini
    • --region - region deployment
    • --allow-unauthenticated - bisa diakses publik
  3. Proses deploy akan:
    • Build Docker image dari source code
    • Upload image ke Google Container Registry
    • Deploy container ke Cloud Run
  4. Tunggu hingga selesai (~3-5 menit untuk pertama kali)

Langkah 5: Akses Aplikasi

  1. Setelah deploy selesai, akan muncul URL seperti: 
    https://hello-cloud-xxxxxxxxxx-as.a.run.app
  2. Copy URL tersebut dan buka di browser
  3. Anda akan melihat pesan "Hello from Cloud Run!"
  4. Selamat! Aplikasi pertama Anda sudah online di cloud üéâ

Langkah 6: Update Aplikasi

  1. Edit app.py, ubah teks di HTML
  2. Deploy ulang dengan command yang sama: 
    gcloud run deploy hello-cloud --source . --region=asia-southeast1 --allow-unauthenticated
  3. Proses lebih cepat karena cache (~1-2 menit)
  4. Refresh browser untuk lihat perubahan

Bagian 6: Monitoring Aplikasi di Cloud Run

Langkah 1: Buka Cloud Run Console

  1. Buka Console: https://console.cloud.google.com/
  2. Menu ‚ò∞ ‚Üí Cloud Run
  3. Klik service hello-cloud

Langkah 2: Lihat Metrics

Di halaman detail service, Anda bisa lihat:

  • Request count - Jumlah request ke aplikasi
  • Request latency - Waktu response
  • Container instance count - Jumlah container running
  • Billable container instance time - Usage untuk billing

Langkah 3: Lihat Logs

  1. Tab LOGS di halaman service
  2. Akan menampilkan semua logs dari aplikasi
  3. Gunakan filter untuk cari logs tertentu
  4. Log juga bisa diakses via command: 
    gcloud run services logs read hello-cloud --region=asia-southeast1

Langkah 4: Cek Revisions

  1. Tab REVISIONS
  2. Setiap deploy membuat revision baru
  3. Anda bisa rollback ke revision sebelumnya jika ada masalah
  4. Traffic bisa di-split antar revisions (untuk A/B testing)

Bagian 7: Kelola Project & Billing

Cek Penggunaan & Cost

  1. Menu ‚ò∞ ‚Üí Billing ‚Üí Reports
  2. Lihat grafik usage dan cost per service
  3. Filter by service untuk lihat cost Cloud Run
  4. Setup budget alerts untuk notifikasi jika cost melebihi threshold
Tips Hemat: Cloud Run free tier mencakup:
  • 2 juta requests per bulan
  • 360,000 GB-seconds of memory
  • 180,000 vCPU-seconds of compute time
Untuk belajar dan proyek kecil, biasanya masuk free tier.

Setup Budget Alert

  1. Menu ‚ò∞ ‚Üí Billing ‚Üí Budgets & alerts
  2. Klik CREATE BUDGET
  3. Set budget amount (misalnya $10)
  4. Set alert thresholds (50%, 90%, 100%)
  5. Tambahkan email untuk notifikasi
  6. Klik FINISH

Bagian 8: Clean Up Resources

Penting: Untuk menghindari charges yang tidak perlu, hapus resources yang tidak digunakan.

Opsi 1: Delete Service

gcloud run services delete hello-cloud --region=asia-southeast1

Opsi 2: Delete Entire Project

  1. Menu ‚ò∞ ‚Üí IAM & Admin ‚Üí Settings
  2. Klik SHUT DOWN
  3. Masukkan Project ID untuk konfirmasi
  4. Klik SHUT DOWN
Catatan: Project yang di-shutdown akan dihapus permanen setelah 30 hari. Dalam periode ini masih bisa di-restore.

Bagian 9: Command-Command Penting gcloud

Konfigurasi & Auth

gcloud auth login                    # Login ke akun Google
gcloud config list                   # Lihat konfigurasi
gcloud config set project PROJECT_ID # Set project aktif
gcloud projects list                 # List semua projects

Cloud Run

gcloud run deploy                    # Deploy service
gcloud run services list             # List services
gcloud run services describe SERVICE # Detail service
gcloud run services delete SERVICE   # Hapus service
gcloud run services logs read SERVICE # Lihat logs

Regions & Zones

gcloud compute regions list          # List regions
gcloud compute zones list            # List zones
gcloud config set compute/region REGION # Set default region

APIs & Services

gcloud services list                 # List enabled APIs
gcloud services enable API_NAME      # Enable API
gcloud services disable API_NAME     # Disable API

Help & Info

gcloud --help                        # General help
gcloud run --help                    # Help untuk Cloud Run
gcloud run deploy --help             # Help untuk specific command
gcloud info                          # Info environment

Deploy Flutter App ke Cloud

Untuk deploy Flutter web app ke GCP:

  1. Build Flutter web: 
    flutter build web
  2. Upload ke Cloud Storage: 
    gsutil -m cp -r build/web/* gs://bucket-name/
  3. Set bucket sebagai website: 
    gsutil web set -m index.html gs://bucket-name/
  4. Set public access: 
    gsutil iam ch allUsers:objectViewer gs://bucket-name

Best Practices untuk Production

Security

  • Jangan expose API keys di source code
  • Gunakan Secret Manager untuk credentials
  • Enable authentication untuk service yang sensitif
  • Setup firewall rules dan VPC

Cost Optimization

  • Set memory & CPU limits sesuai kebutuhan
  • Gunakan min instances = 0 untuk development
  • Monitor usage dan set budget alerts
  • Hapus resources yang tidak terpakai

Performance

  • Pilih region terdekat dengan users
  • Optimize Docker image size
  • Enable CDN untuk static assets
  • Setup auto-scaling yang tepat

Monitoring & Logging

  • Setup Cloud Monitoring untuk alerts
  • Gunakan structured logging
  • Setup error reporting
  • Monitor latency dan error rates

Troubleshooting Umum

Error: Billing not enabled

Solusi: Link billing account ke project di Console ‚Üí Billing

Error: API not enabled

Solusi: Jalankan gcloud services enable API_NAME

Deploy gagal: Permission denied

Solusi: Pastikan sudah login dengan gcloud auth login dan project sudah di-set

Container tidak bisa start

Solusi: Cek logs untuk error. Pastikan app listen di port dari env var PORT

URL tidak bisa diakses

Solusi: Pastikan deploy dengan flag --allow-unauthenticated
